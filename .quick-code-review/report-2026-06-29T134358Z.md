
## Adversarial-Codex

### [important] Unexecuted non-Vitest tests are reported as passing gate evidence

**`src/verifier/deterministic/strategies/test.ts:21-33`** — _Codex_ · severity: high · confidence: 0.92

The strategy filters out pgTAP/Go/etc. test files and either (a) runs only the JS/TS subset (mixed diff) or (b) returns `ran("test", true, ...)` without invoking any test runner (pure non-JS diff). Both paths violate the gate contract that `ran` means "the check executed and observed a pass signal."

**Mixed diff:** a broken `.test.sql` or `_test.go` is silently ignored if the JS/TS subset passes. **Pure non-JS diff:** the merge gate receives `observed:true` even though nothing executed. If external CI is missing, delayed, or not equivalent, broken database/Go/etc. tests can ship.

**Recommendation (Codex):** Do not emit passing `GateRan` evidence for files this strategy did not execute. Either run a configured non-Vitest test command for those files, or fail/skip explicitly with a reason that cannot be mistaken for ground-truth pass evidence.

**Note on design intent:** The implementation comment (test.ts:22-25) explicitly delegates non-JS green-ness to the reviewer panel + target-repo CI. Codex is disputing that design choice. The finding is substantively valid — `ran(true)` is misleading when no runner executed — but the fix may be as simple as a distinct outcome (e.g. `ran("test", true, ..., { executed: false })`) rather than wiring non-JS runners.
