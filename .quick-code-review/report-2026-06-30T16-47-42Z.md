#### [minor] `src/cli/subcommands/run.ts:61` — run subcommand imports a constant from its peer scaffold subcommand

- **Reviewer**: architecture
- **Quote**: `import { E2E_CHECK_CONTEXT } from "./scaffold.js";`
- **Why**: Sibling-to-sibling edge between two CLI subcommands created solely to share the one-line constant `E2E_CHECK_CONTEXT` (`"E2E"`). It pulls the whole scaffold module (and its git/config/gate-env transitive deps) into `run.ts`'s import graph for a value the run path only uses to filter staging required checks. `madge` confirms NO cycle and the value's authority is arguably the scaffold (it must match `e2e.yml`'s `name: E2E`), so it's defensible today — but a zero-dep shared-names leaf already exists (`src/shared/gate-config-names.ts`) for exactly this class of constant, and the next scaffold-owned token will widen this peer edge.
- **Fix sketch**: Relocate `E2E_CHECK_CONTEXT` to a zero-dependency shared leaf and have both subcommands import it downward; keep a comment tying it to `e2e.yml`'s job name.

### Security

#### [minor] `templates/tests/e2e/auth.setup.ts:20` — scaffolded auth.setup.ts streams the auth command's output to CI logs + the uploaded report (CWE-532)

- **Reviewer**: security
- **Quote**: `execSync(cmd, { stdio: "inherit" });`
- **Why**: Source `const cmd = __E2E_AUTH_SETUP_COMMAND__;` (line 18) is the project's `e2e.authSetupCommand`, run in the authenticated nightly suite. `stdio:"inherit"` pipes that command's stdout/stderr into the GitHub Actions job log AND captures it as the Playwright `setup` test output inside `playwright-report/`, which `templates/ci/e2e-nightly.yml` uploads unconditionally (`if: ${{ !cancelled() }}`, 7-day retention). Auth/login CLIs commonly echo a bearer/session token or "logged in as <user>", which would then land in a build log + a downloadable artifact readable by every collaborator (public for public repos). Real but bounded — depends on the project's own command printing secrets, and mitigations are strong (`trace:"off"` on setup+semantic projects, `tests/e2e/.auth/` gitignored, fail-loud storageState verification).
- **Fix sketch**: Capture the command output (`pipe`/redirect) instead of inheriting it, or document in the template that `authSetupCommand` must not print secrets to stdout/stderr.

### Systemic

#### [important] `src/orchestrator/finalize.ts:223` — session crash mid-ADJUDICATE permanently fails an otherwise-complete run; documented resume recovery is unreachable

- **Reviewer**: systemic-failure
- **Failure mode**: `invariant-without-repair`
- **Scenario**: A session run finishes all tasks, the sweep flags a blocking money-path, the runner fixes it forward and commits to staging, but the process dies before `factory run e2e --adjudicated`; on resume the sweep stage never re-enters (`wantsSemanticSweep` returns false for a session `done` marker — re-entry gated to `mode==="workflow"`), so the backstop forces the complete, fixed run to terminal `failed` with no path to record the adjudication and ship.
- **Quote**: `terminal = "failed";`
- **Chain anchors**:
  - `src/orchestrator/finalize.ts:223` — `terminal = "failed";` _(actual outcome — complete+fixed session run forced to absorbing failed)_
  - `src/orchestrator/finalize.ts:218` — `Resume re-enters ADJUDICATE, which records the marker.` _(promised-but-unreachable repair path)_
  - `src/orchestrator/next.ts:129` — `run.mode === "workflow" &&` _(re-sweep/ADJUDICATE re-entry gated to workflow only)_
- **Why**: The finalize backstop and SKILL.md both assert the lost `adjudicated` marker is repaired by resume re-entering ADJUDICATE, but no engine path makes that re-entry possible for a session-mode `done` sweep — `next.ts` returns false before the workflow-only re-sweep branch, the runner only ever receives a `finalize` envelope, and `factory run finalize` drives the run to an absorbing `failed`. The invariant "a blocking session sweep carries a durable adjudication record" has no convergence/repair when the record is lost to a crash; only recovery is discarding all completed work and re-running from scratch.
- **Fix sketch**: On resume, let a session run with `e2e_sweep done + blocking_count>0 + !adjudicated` re-enter ADJUDICATE (emit an adjudicate-needed envelope instead of finalize); only fall through to terminal `failed` once the cap is exhausted.

#### [important] `src/orchestrator/e2e-sweep.ts:274` — workflow money-path suspend is quota-gated at resume, violating workflow's no-quota contract and stalling the re-verify loop

- **Reviewer**: systemic-failure
- **Failure mode**: `invariant-without-repair`
- **Scenario**: A workflow run (quota pacing disabled by contract) completes all tasks, the sweep blocks on a money-path and suspends with no quota checkpoint; the user fixes the money-path on staging and runs `/factory:resume`, but `planResume` runs the live pacer (short-circuits only for `ignore_quota`, never for `mode==="workflow"`), so an over-curve reading returns pause/suspend-7d or an unobservable reading returns `unavailable-halt` and refuses resume — parking the re-sweep behind a quota window the workflow run was never subject to.
- **Quote**: `const suspendOnBlocking = run.mode === "workflow" && blockingCount > 0;`
- **Chain anchors**:
  - `src/orchestrator/e2e-sweep.ts:274` — `const suspendOnBlocking = run.mode === "workflow" && blockingCount > 0;` _(introduces the non-quota workflow money-path suspend)_
  - `src/orchestrator/quota-gate.ts:43` — `if (mode === "workflow" || ignoreQuota) return null;` _(drive-loop honors the workflow no-quota contract)_
  - `src/quota/resume.ts:63` — `const decision = evaluatePacer(reading, config, nowEpoch);` _(resume gates ALL suspended runs on the live pacer regardless of mode — only ignore_quota exempted)_
- **Why**: The drive-loop quota gate explicitly skips workflow runs (they can't observe usage and opted out of pacing), but the resume seam applies `evaluatePacer` to every non-`ignore_quota` suspended run regardless of mode. This change introduces a prominent new non-quota workflow suspend whose entire recovery story ("fix on staging, then resume — the re-sweep re-verifies") routes through that resume seam, so the workflow no-quota invariant is silently unenforced exactly where the feature depends on it.
- **Fix sketch**: Mirror `quota-gate.ts` in `planResume`: short-circuit to resume for `mode==="workflow"` (or distinguish a non-quota money-path suspend from a quota suspend and only pace the latter).

#### [important] `src/orchestrator/next.ts:120-135` — session resume bypasses E2E adjudication and terminally fails recoverable runs

- **Reviewer**: codex-adversarial
- **Codex severity**: high · **Confidence**: 0.89
- **Quote**: _n/a — existence-checked (review schema has no verbatim field)_
- **Why**: For a session run with `e2e_sweep.status === "done"` and `blocking_count > 0`, `wantsSemanticSweep` always returns false. On a crash after `runE2eRecord` records the blocking sweep but before the runner calls `--adjudicated`, `nextTask` returns `finalize` instead of re-entering the semantic/adjudication stage, and `finalizeRun` then marks the otherwise-completed run `failed` because the adjudication marker is absent — making a recoverable crash a terminal failure, despite runner instructions saying resume should re-run adjudication. _(Same root cause as Systemic #1 — see Themes.)_
- **Fix sketch**: Teach `nextTask` to surface a resumable semantic/adjudication step for session runs with `done + blocking_count>0 + !adjudicated`, or add a separate engine envelope for pending adjudication so resume cannot go straight to finalize.

#### [important] `src/config/schema.ts:213-300` — empty `e2e.startCommand` validates but disables scaffolding (gate disappears silently)

- **Reviewer**: codex-adversarial
- **Codex severity**: high · **Confidence**: 0.92
- **Quote**: _n/a — existence-checked (review schema has no verbatim field)_
- **Why**: `startCommand` is `z.string().optional()` and the refinement only rejects `undefined`, so `""` satisfies `e2e.semantic` / `e2e.blocking`. Scaffold gates all E2E template generation + required-check insertion on truthiness (`if (opts.config.e2e.startCommand)`), so an opted-in blocking smoke gate can validate while producing no `playwright.config.ts`, no E2E workflow, and no required `E2E` check. The orchestrator uses a different presence check (`!== undefined`), so semantic runs may still be scheduled with an empty boot command — a gate disappearing silently under a realistic misconfiguration.
- **Fix sketch**: Normalize/validate `startCommand` as a non-empty trimmed string and use one shared predicate in config validation, scaffold, and `wantsSemanticSweep`; add regression tests for `""` and whitespace-only values with `semantic`/`blocking` enabled.
