### [minor] templates/tests/e2e/auth.setup.ts:20 — Scaffolded auth.setup.ts streams the auth command's output to CI logs and the uploaded report (stdio:"inherit") — CWE-532 token-leak surface
- kind: local
- quote: `execSync(cmd, { stdio: "inherit" });`
- why: Source: line 18 `const cmd = __E2E_AUTH_SETUP_COMMAND__;` is the project's e2e.authSetupCommand, run in the authenticated nightly suite. Sink: execSync with stdio:"inherit" pipes that command's stdout/stderr into the GitHub Actions job log AND captures them as the Playwright `setup` test's output inside playwright-report/, which templates/ci/e2e-nightly.yml uploads unconditionally (`if: ${{ !cancelled() }}`, retention-days:7). Auth/login CLIs commonly echo a bearer/session token or a 'logged in as <user>' line, so any such output lands in a build log and a downloadable artifact readable by every repo collaborator (and publicly for public repos). Real but bounded: exposure depends on the project's own command printing secrets, and existing mitigations are strong (trace:"off" on the setup+semantic projects so no request/header trace is captured, tests/e2e/.auth/ gitignored, fail-loud storageState verification). Hardening: capture the command output (redirect/'pipe') instead of inheriting it, or document in the template that authSetupCommand must not print secrets to stdout/stderr. Severity LOW for those reasons.
