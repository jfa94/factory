# Rescue Command Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build `/factory:rescue` — a complex-recovery command that scans a pipeline run's state and GitHub artifacts, auto-applies safe fixes, batch-approves risky/destructive ones, dispatches a read-only diagnostic subagent for failed tasks, applies the resulting plans, and hands off to `/factory:run resume`. Also standardize task PR title/commit format to enable GitHub-side discovery.

**Architecture:** Deterministic-first. Scan (`pipeline-rescue-scan`) and apply (`pipeline-rescue-apply`) are bash scripts; a thin skill orchestrates them plus `AskUserQuestion` batch approvals and parallel `Agent()` dispatches for diagnosis. A new `rescue-diagnostic` agent (Sonnet, read-only) produces structured JSON that maps to deterministic actions in apply. All state writes route through `pipeline-state`.

**Tech Stack:** Bash, jq, gh CLI, git. No new language or runtime dependencies. Markdown (skills, agents, command, refs). Tests use the existing bash-test pattern in `bin/tests/` with `gh` mocked via PATH shim.

**Reference spec:** `docs/superpowers/specs/2026-04-24-rescue-design.md`.

```bash
_default_task_commit_message() {
  local tid="$1" rid="$2" state issue desc fallback
  fallback="chore(factory): finalize task ${tid}"
  if [[ -z "$rid" ]]; then
    # Fall back to 'current' symlink
    if [[ -L "$CLAUDE_PLUGIN_DATA/runs/current" ]]; then
      rid=$(basename "$(readlink "$CLAUDE_PLUGIN_DATA/runs/current")")
    fi
  fi
  [[ -z "$rid" ]] && { printf '%s' "$fallback"; return; }
  state="$CLAUDE_PLUGIN_DATA/runs/$rid/state.json"
  [[ -f "$state" ]] || { printf '%s' "$fallback"; return; }
  issue=$(jq -r '.input.issue_numbers[0] // empty' "$state" 2>/dev/null)
  desc=$(jq -r --arg t "$tid" '.tasks[$t].description // empty' "$state" 2>/dev/null)
  if [[ -z "$issue" || -z "$desc" ]]; then
    printf '%s' "$fallback"
    return
  fi
  # Truncate description to 72 chars
  if (( ${#desc} > 72 )); then
    desc="${desc:0:72}"
  fi
  printf '[%s] task(%s): %s' "$issue" "$tid" "$desc"
}
```

```bash
#!/usr/bin/env bash
# Shared helpers for pipeline-rescue-scan and pipeline-rescue-apply.
# Source (do not execute): source "$(dirname "$0")/pipeline-rescue-lib.sh"

# shellcheck disable=SC2034

rescue_now_iso() {
  date -u +%Y-%m-%dT%H:%M:%SZ
}

# Return the parent PRD issue number for a run id, empty string if unknown.
rescue_issue_for_run() {
  local run_id="$1" state
  state="$CLAUDE_PLUGIN_DATA/runs/$run_id/state.json"
  [[ -f "$state" ]] || { printf ''; return; }
  jq -r '.input.issue_numbers[0] // empty' "$state" 2>/dev/null
}

# Return 0 if a PR title belongs to the given issue, 1 otherwise.
rescue_pr_belongs_to_run() {
  local title="$1" issue="$2"
  [[ "$title" == "[$issue] task("* ]]
}

# Parse task_id out of a standardized PR title; empty on no match.
rescue_task_id_from_title() {
  local title="$1"
  if [[ "$title" =~ ^\[[0-9]+\]\ task\(([^)]+)\): ]]; then
    printf '%s' "${BASH_REMATCH[1]}"
  fi
}

# Write a structured audit entry to .rescue.applied_actions[].
rescue_audit() {
  local run_id="$1" phase="$2" issue_id="$3" task_id="$4" action="$5" result="$6" error="${7:-}"
  local entry
  entry=$(jq -n \
    --arg ts "$(rescue_now_iso)" \
    --arg phase "$phase" \
    --arg issue_id "$issue_id" \
    --arg task_id "$task_id" \
    --arg action "$action" \
    --arg result "$result" \
    --arg error "$error" \
    '{ts: $ts, phase: $phase, issue_id: $issue_id, task_id: $task_id, action: $action, result: $result, error: (if $error == "" then null else $error end)}')
  # Append via jq; keep state writes through pipeline-state.
  local current new
  current=$(pipeline-state read "$run_id" '.rescue.applied_actions // []' 2>/dev/null || echo '[]')
  new=$(jq --argjson entry "$entry" '. + [$entry]' <<<"$current")
  pipeline-state write "$run_id" '.rescue.applied_actions' "$new" >/dev/null
}
```

Create `bin/pipeline-rescue-scan`:

```bash
#!/usr/bin/env bash
# Scan a pipeline run for rescue-actionable issues.
# Usage: pipeline-rescue-scan <run-id>
# Output: JSON on stdout with {run_id, state_summary, mechanical_issues[], investigation_flags[]}
# Exit: 0 on success (including "no issues"), 1 on fatal (state missing, GitHub unreachable).

set -euo pipefail

source "$(dirname "$0")/pipeline-lib.sh"
source "$(dirname "$0")/pipeline-rescue-lib.sh"
require_command jq

run_id="${1:?missing run-id}"
state="$CLAUDE_PLUGIN_DATA/runs/$run_id/state.json"
[[ -f "$state" ]] || { log_error "no state for run $run_id"; exit 1; }

mechanical=()
investigation=()
state_summary=$(jq '{status: .status, tasks: (.tasks // {} | to_entries | map({task_id: .key, status: .value.status, stage: .value.stage, pr_number: .value.pr_number}))}' "$state")

# I-01: stale state lock
lock_dir="$CLAUDE_PLUGIN_DATA/runs/$run_id/state.lock"
if [[ -d "$lock_dir" ]]; then
  pid_file="$lock_dir/pid"
  if [[ -f "$pid_file" ]]; then
    holder=$(<"$pid_file")
    if [[ -n "$holder" ]] && ! kill -0 "$holder" 2>/dev/null; then
      mechanical+=("$(jq -n --arg id I-01 --argjson tier 1 --arg desc "stale state lock (dead PID $holder)" '{id: $id, tier: $tier, task_id: null, description: $desc}')")
    fi
  fi
fi

# Get task entries for PR-related checks
tasks_json=$(jq '.tasks // {}' "$state")
issue=$(rescue_issue_for_run "$run_id")

# For each task with a PR, fetch PR state via gh
while IFS=$'\t' read -r task_id pr_number status stage ci_status; do
  [[ -z "$task_id" ]] && continue
  [[ -z "$pr_number" || "$pr_number" == "null" ]] && continue
  pr_json=$(gh pr view "$pr_number" --json state,mergedAt,mergeable,statusCheckRollup 2>/dev/null || echo '{}')
  pr_state=$(jq -r '.state // ""' <<<"$pr_json")

  # I-03: PR merged, state not updated
  if [[ "$pr_state" == "MERGED" && "$status" != "done" ]]; then
    mechanical+=("$(jq -n --arg id I-03 --argjson tier 1 --arg task "$task_id" --arg desc "PR #$pr_number merged but task status=$status" '{id: $id, tier: $tier, task_id: $task, description: $desc}')")
  fi

  # I-05: stale CI status (only if stage=ship; others don't care)
  latest_ci=$(jq -r '[.statusCheckRollup[]?.conclusion] | map(select(. != null)) | if length == 0 then "" elif all(. == "SUCCESS") then "green" else "red" end' <<<"$pr_json")
  if [[ "$stage" == "ship" && -n "$latest_ci" && -n "$ci_status" && "$ci_status" != "null" && "$ci_status" != "$latest_ci" ]]; then
    mechanical+=("$(jq -n --arg id I-05 --argjson tier 1 --arg task "$task_id" --arg desc "ci_status=$ci_status but PR shows $latest_ci" '{id: $id, tier: $tier, task_id: $task, description: $desc}')")
  fi
done < <(jq -r '.tasks // {} | to_entries[] | [.key, (.value.pr_number // ""), (.value.status // ""), (.value.stage // ""), (.value.ci_status // "")] | @tsv' "$state")

# I-04: PR exists on GitHub (via title search) but state.pr_url empty
if [[ -n "$issue" ]]; then
  search_json=$(gh pr list --search "[$issue] task( in:title" --state all --json number,title,state,mergedAt,mergeable,headRefName,url 2>/dev/null || echo '[]')
  while IFS=$'\t' read -r pr_num title; do
    [[ -z "$pr_num" ]] && continue
    tid=$(rescue_task_id_from_title "$title")
    [[ -z "$tid" ]] && continue
    state_pr=$(jq -r --arg t "$tid" '.tasks[$t].pr_url // ""' "$state")
    if [[ -z "$state_pr" ]]; then
      # Verify the task exists in state before flagging
      exists=$(jq -r --arg t "$tid" '.tasks[$t].task_id // ""' "$state")
      if [[ -n "$exists" ]]; then
        mechanical+=("$(jq -n --arg id I-04 --argjson tier 1 --arg task "$tid" --arg desc "PR #$pr_num exists on GitHub but state.pr_url empty" '{id: $id, tier: $tier, task_id: $task, description: $desc}')")
      fi
    fi
```

```bash
# Apply rescue remediations. Idempotent per action.
# Usage:
#   pipeline-rescue-apply --tier=safe --plan=<report.json>
#   pipeline-rescue-apply --tier=risky --plan=<approved.json>
#   pipeline-rescue-apply --plans=<approved-plans.json>        (investigation)
#   pipeline-rescue-apply --dry-run ...
# Exit: 0 on success (even with per-action errors logged), 1 on fatal.

set -euo pipefail

source "$(dirname "$0")/pipeline-lib.sh"
source "$(dirname "$0")/pipeline-rescue-lib.sh"
require_command jq

tier=""
plan=""
plans=""
dry_run=0

while [[ $# -gt 0 ]]; do
  case "$1" in
    --tier) tier="$2"; shift 2 ;;
    --tier=*) tier="${1#--tier=}"; shift ;;
    --plan) plan="$2"; shift 2 ;;
    --plan=*) plan="${1#--plan=}"; shift ;;
    --plans) plans="$2"; shift 2 ;;
    --plans=*) plans="${1#--plans=}"; shift ;;
    --dry-run) dry_run=1; shift ;;
    *) log_error "unknown flag: $1"; exit 1 ;;
  esac
done

if [[ -z "$plan" && -z "$plans" ]]; then
  log_error "must pass --plan or --plans"
  exit 1
fi

_apply_I_03() {
  local run_id="$1" task_id="$2"
  local cur
  cur=$(pipeline-state read "$run_id" ".tasks.$task_id.status" 2>/dev/null || echo '""')
  if [[ "$cur" == '"done"' ]]; then
    rescue_audit "$run_id" safe I-03 "$task_id" mark_pr_merged skipped ""
    return 0
  fi
  if (( dry_run )); then
    rescue_audit "$run_id" safe I-03 "$task_id" mark_pr_merged dry_run ""
    return 0
  fi
  pipeline-state task-status "$run_id" "$task_id" done >/dev/null
  pipeline-state task-write "$run_id" "$task_id" stage '"ship_done"' >/dev/null
  rescue_audit "$run_id" safe I-03 "$task_id" mark_pr_merged ok ""
}

_apply_I_01() {
  local run_id="$1"
  local lock="$CLAUDE_PLUGIN_DATA/runs/$run_id/state.lock"

}

_apply_I_02() {

  rescue_audit "$run_id" safe I-02 "" remove_orphan_worktree ok "$wt_path"
}

_apply_I_04() {
  local run_id="$1" task_id="$2" pr_url="$3"
  local cur
  cur=$(pipeline-state read "$run_id" ".tasks.$task_id.pr_url" 2>/dev/null || echo '""')
  if [[ "$cur" != '""' && "$cur" != "null" ]]; then
    rescue_audit "$run_id" safe I-04 "$task_id" backfill_pr_url skipped ""
    return 0
  fi
  local pr_num
  pr_num=$(printf '%s' "$pr_url" | grep -oE '[0-9]+$' || printf '')
  pipeline-state task-write "$run_id" "$task_id" pr_url "\"$pr_url\"" >/dev/null

_dispatch_safe() {

      I-03) _apply_I_03 "$run_id" "$task_id" ;;

  _dispatch_safe "$run_id" "$plan"
elif [[ "$tier" == "risky" ]]; then
  log_warn "risky-tier apply not implemented in this task; no-op"
fi

printf '{"run_id":"%s","tier":"%s"}\n' "$run_id" "${tier:-plans}"
```

```markdown
1. Never edit `state.json` directly.
2. Never attempt fixes not covered by `pipeline-rescue-apply`.
3. Always run `pipeline-ensure-autonomy` first.
4. Tier-1 fixes auto-apply without prompting.
5. Tier-2 and tier-3 fixes require `AskUserQuestion` batch approval.
6. The `rescue-diagnostic` agent is read-only; its decisions drive deterministic apply actions.
7. Final step is always `/factory:run resume`, unless the user cancels.

## Protocol

1. **Autonomy check.** Run `pipeline-ensure-autonomy`. If it exits 2 (stale or missing), follow the existing relaunch prompt pattern from `/factory:run`.

3. **Refuse if pipeline is live.** If `.status == "running"` and a fresh `pipeline-state` lock attempt succeeds (meaning no current holder), proceed. Otherwise exit with "Pipeline is running; wait for it to halt before rescuing."

4. **Scan.** Run `pipeline-rescue-scan <run-id> > $rundir/rescue/report-<ts>.json`. Read the report.

5. **Short-circuit if clean.** If `mechanical_issues` and `investigation_flags` are both empty AND no task is `status=failed`, skip to step 12 (invoke resume).

6. **Auto-apply tier-1.** Run `pipeline-rescue-apply --tier=safe --plan=<report>`. Silent.

7. **Mechanical batch approval.** Collect all tier-2 and tier-3 mechanical issues. If any exist, call `AskUserQuestion` with one question listing them and three options: `approve-all`, `review-per-item`, `cancel`.

   - On `review-per-item`: loop, one `AskUserQuestion` per issue with `approve`/`skip`; aggregate approved into the file.
   - On `cancel`: exit with "Rescue cancelled by user."

8. **Apply approved mechanical.** Run `pipeline-rescue-apply --tier=risky --plan=<approved-mechanical>.json`.

9. **Investigation.** Read the current state; for every task where `.tasks.<id>.status == "failed"` OR every `investigation_flags[]` entry, build an input JSON at `$rundir/rescue/diagnostic.<task-id>.input.json` (see `reference/diagnostic-agent-contract.md`).     - On `review-per-item`: loop per plan.
    - On `cancel`: exit.

12. **Invoke resume.** Skill complete — hand control back by instructing the user (or downstream orchestrator) to run `/factory:run resume`. If operating in autonomous mode where you can directly invoke, call the `pipeline-orchestrator` skill with `mode=resume`.
```
